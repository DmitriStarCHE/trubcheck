# Шаблоны труб — Спецификация

**Дата:** 2026-04-20  
**Проект:** Pipe Tracker Pro (TrubCheck)  
**Статус:** Согласовано, готово к реализации

---

## Цель

Ускорить создание документов: пользователь один раз сохраняет набор типоразмеров трубы (ГОСТ + Ø + марка стали) как шаблон, и в следующий раз применяет его в один тап — форма заполняется без повторного ввода спецификаций.

---

## Что шаблон содержит и что НЕ содержит

| Входит в шаблон | НЕ входит (вводится вручную каждый раз) |
|---|---|
| Название шаблона | Длины отдельных труб |
| ГОСТ (10704, 8732...) | Кол-во и тоннаж пачек |
| Диаметр × толщина | Контрагент |
| Марка стали | Водитель / авто |
| Режим (по трубам / пачками) | Место, дата, фото, примечание |

Шаблон при применении создаёт **пустые карточки** типов трубы с заполненными спецификациями. Длины/пачки всегда разные — пользователь вводит их сам.

## Архитектура

### Новый store в IndexedDB (db/index.js, версия 2)

```js
// templates store
{
  id: string,          // randomUUID
  name: string,        // "Стандарт 57×3"
  pipeTypes: [
    {
      gost: string,
      diameter: string,
      thickness: string,
      steelGrade: string,
      mode: 'individual' | 'batch'
    }
  ],
  createdAt: string    // ISO
}
```

Новые функции в `db/index.js`:
- `getAllTemplates()` — все шаблоны
- `saveTemplate(tpl)` — создать / обновить
- `deleteTemplate(id)` — удалить

Миграция: `DB_VERSION` → 2, в `upgrade()` добавить `templates` store при `oldVersion < 2`.

## Новые файлы

### `src/pages/TemplatesPage.jsx`

Структура страницы:
- Заголовок + кнопка «+ Новый шаблон»
- Список шаблонов (карточки) — название + перечень типоразмеров
- На каждой карточке: кнопка «Изменить» и «Удалить»
- Пустое состояние: иконка + «Шаблонов пока нет»
- Форма создания/редактирования — inline-панель под кнопкой «+ Новый» (без отдельной страницы):
  - Поле «Название»
  - Блок добавления типов трубы (те же поля что в AccountingPage: ГОСТ, Ø, толщина, марка, режим)
  - Кнопки «Сохранить» / «Отмена»

## Изменяемые файлы

### `src/db/index.js`
- Поднять `DB_VERSION` до 2
- В `upgrade()` добавить `templates` store с `keyPath: 'id'`
- Добавить `getAllTemplates`, `saveTemplate`, `deleteTemplate`

### `src/router.jsx`
```jsx
{ path: 'templates', element: <Suspense ...><TemplatesPage /></Suspense> }
```

### `src/App.jsx`
- 4-я иконка в навбаре: «Шаблоны» (иконка звезды или закладки)
- `NavLink to="/templates"`

### `src/pages/AccountingPage.jsx`

**Кнопка «Применить шаблон»** — размещается над блоком типов трубы:
- Открывает модальный bottom sheet со списком шаблонов
- При выборе шаблона: текущие `pipeTypes` **заменяются** на пустые копии из шаблона (lengths: [emptyLength()], batches: [emptyBatch()])
- Если шаблонов нет — кнопка ведёт на страницу Шаблонов

**Кнопка «Сохранить как шаблон»** — размещается под списком типов трубы:
- Активна только если есть хотя бы один тип трубы с заполненным диаметром
- Показывает inline-поле для ввода названия + кнопку «Сохранить»
- После сохранения показывает toast «Шаблон сохранён»

## UX-детали

- Bottom sheet для выбора шаблона — список с именем и кратким перечнем типоразмеров
- Замена pipe types происходит без подтверждения (пользователь выбрал сознательно)
- Если форма непустая — показать confirm «Заменить текущие трубы шаблоном?»
- Шаблоны хранятся локально, синхронизации нет

## Что НЕ входит в скоуп

- Шаблоны для контрагентов / водителей
- Импорт/экспорт шаблонов
- Сортировка и группировка шаблонов
